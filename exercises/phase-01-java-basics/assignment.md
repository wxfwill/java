# 权限菜单程序练习任务

## 一、对象模型

自行设计以下对象，不提供字段答案：

- 用户
- 角色
- 菜单
- 用户角色关系
- 角色菜单关系

设计前回答：

- 哪些属性决定对象身份？
- 哪些属性可变？
- 哪些对象适合使用 Record，哪些不适合？
- 是否允许重复角色和重复菜单？

## 二、菜单能力

实现：

- 根据用户查询角色
- 根据角色查询菜单
- 菜单去重
- 菜单排序
- 按类型分组
- 构建菜单树
- 按 ID 递归查找菜单
- 过滤禁用菜单

## 三、异常与文件

实现：

- 用户不存在异常
- 角色不存在异常
- 菜单关系非法异常
- CSV 用户、角色或菜单导入
- 空文件、非法列、重复数据和编码异常处理

## 四、测试要求

至少自行设计 20 个测试，覆盖：

- 正常用户权限
- 无角色用户
- 多角色重复菜单
- 空菜单集合
- 多层菜单树
- 孤立子菜单
- 菜单循环关系
- 禁用菜单
- 重复 ID
- `equals` 与 `hashCode`
- 空 CSV
- 非法 CSV 列
- 重复 CSV 数据
- 不存在的用户
- 不存在的角色
- 排序稳定性
- 分组结果
- 递归查找成功
- 递归查找失败
- 不可变对象边界

测试名称必须体现业务场景，不使用 `test1`、`test2` 等无意义名称。
