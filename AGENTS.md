# Java Full-Stack Learning Workspace Guide

## 仓库目标
- 本仓库用于前端架构师向 Java 全栈开发转型的长期学习、练习和主线项目。
- 新项目以 Java 21、Spring Boot 3.x、Maven、MySQL 8、Redis 为主线，同时培养 Java 8/11 与 Spring Boot 2.x 存量项目阅读能力。
- 学习周期默认 48 周，每周投入 4–6 小时，以阶段验收结果而不是周数决定是否进入下一阶段。

## 学习边界
- 前 16 周以手写、调试和测试为主，AI 不代写核心方法、完整 SQL 或练习答案。
- 第 17–34 周可使用 AI 审查实现、补充测试场景和解释调用链，但数据模型、API、事务、权限、Token、缓存与幂等方案必须由学习者先设计。
- 第 35 周以后可使用 AI 生成重复骨架和辅助重构，但安全、权限、事务、并发、数据迁移与架构选型始终由人工决策。
- 未完成模块化单体项目之前，不引入完整微服务、Spring Cloud、Kubernetes 或非必要消息队列。

## 代码要求
- Java 代码使用 Java 21 语法基线，公共类型、方法和配置必须有明确意图。
- 练习代码优先可读性，不为减少行数滥用 Lombok、反射、代码生成器或复杂设计模式。
- Controller 不写数据库逻辑和事务；Entity 不直接作为公共请求或响应模型。
- 所有写操作必须考虑参数校验、事务、重复提交、并发、失败恢复与权限边界。
- 新增或修改方法时补充简短注释，说明关键意图或边界，避免空泛说明。

## 验证要求
- 每个练习至少包含正常、非法、边界和失败场景。
- 优先运行目标模块测试，再运行更大范围验证。
- 涉及数据库行为时逐步过渡到 Testcontainers，不用 H2 代替所有真实数据库验证。
- 遇到失败先阅读异常、定位自身代码栈、检查输入/配置/SQL并构造最小复现，再查询资料或询问 AI。

## 仓库协作
- 不提交 IDE 缓存、构建产物、日志、密钥、Token、数据库数据目录或本机环境配置。
- 不修改与当前学习阶段无关的模块，不提前生成后续阶段的完整业务答案。
- 每两周至少形成一个可运行功能、一组测试或一份故障复盘。
